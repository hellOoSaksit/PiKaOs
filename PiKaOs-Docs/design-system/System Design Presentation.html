System · HERMES SA

HERMES — supervisor / worker

แตก quest เป็น DAG ของ subtask แล้วมอบหมายหลาย agent. เป็น reactive state-machine — 3 job เล็กขับเคลื่อน state ที่เก็บใน Postgres

เขียว = ทำแล้วไม่มีสี = ยังไม่ทำ(ส่วนนี้ยังไม่ทำทั้งหมด)
① hermes_plan(orch_id)
โหลด quest + agent ว่าง → LLM แตกเป็น subtasks[] {title, assignee, deps, brief} → เขียน DAG + brief doc → spawn subtask ที่ deps ครบ → exit
▼
② agent_run(child_id)
loop §5 ของ agent ที่รับมอบหมาย → เสร็จ → enqueue hermes_advance
▼
③ hermes_advance(orch_id) · ทุกครั้งที่ลูกเสร็จ
mark subtask done → spawn subtask ที่เพิ่ง deps ครบ → ถ้าหมด → hermes_finalize
▼
④ hermes_finalize(orch_id)
LLM สังเคราะห์ผลรวมจาก output ลูก → orchestration run = done
brief = subtask doc worklog = run_steps subtask → room queue waiting_input → notification (human-in-the-loop) fail → retry N → partial
อ่านกราฟ: HERMES = หัวหน้างาน — แตกงานใหญ่เป็นงานย่อย (DAG) มอบหมายหลาย agent ทำขนานกัน. เป็นแบบ reactive — ทุกครั้งที่ลูกเสร็จจะ trigger ให้แจกงานถัดไปเอง (ไม่มีใครยืนรอ) จึงทน restart และ scale ได้
